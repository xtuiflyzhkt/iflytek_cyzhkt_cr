使用高通9008，MTK工程模式，麒麟工程模式解fblock(bootloader锁)进行刷机，没有bl锁忽略此行
测试机已通过将测试点接地的方式进入麒麟工程模式并成功刷机
推荐在XDA论坛找方法
懂点linux和android内核的可以像我们一样做双空间(但是肯定不完美)，没双空间的可以完整刷机之后安装我们提取的软件
我们的双空间方案通过/data下目录改名实现，类似于备份和恢复备份，示例(建议写shell脚本文件):
cp /data/app /data/app_tmp && cp /data/app_backup /data/app && cp /data/app_tmp /data/app_backup
cp /data/data /data/data_tmp && cp /data/data_backup /data/data && cp /data/data_tmp /data/data_backup
cp /data/media /data/media_tmp && cp /data/media_backup /data/media && cp /data/media_tmp /data/media_backup
cp /data/themes /data/themes_tmp && cp /data/themes_backup /data/themes && cp /data/themes_tmp /data/themes_backup
cp /data/system /data/system_tmp && cp /data/system_backup /data/system && cp /data/system_tmp /data/system_backup
另外，在测试机/data/hw_init/version/special_cust/<测试机型号>/all/cn/app/发现了预装的app：
AppStore,iFlyIME,iflylauncher3,iFlyLogin,mdmadmin,mdmservice,mdmsetting 共7个(不同设备可能不一样)
你需要在进入第二空间的脚本中对这个目录也做处理(移动到其他位置)
建议刷twrp然后重启recovery执行脚本，刷入twrp可以用fastboot
twrp内刷镜像:(简体中文)高级-终端命令:
dd if=<源文件名> of=<目标文件名>
示例:类似于刷回原厂recovery(实际将erecovery复制到recovery):dd if=/dev/block/bootdevice/by-name/erecovery_ramdisk of=/dev/block/bootdevice/by-name/recovery_ramdisk
可以使用twrp-安装刷入Magisk root设备
twrp部分挂载点:
data /data    system /system    SD卡 /external_sd    USB /usb-otg
